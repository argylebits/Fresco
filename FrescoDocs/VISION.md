# Fresco — Vision & Roadmap

## What it is

Fresco is a tool for generating a fresh AI image every day and publishing it to a stable URL. Any project — a GitHub README, a website, a Shopify store — can drop in that URL and get an image that updates itself daily without any manual work.

The name comes from the art form. A fresco is painted fresh each day on wet plaster — the artist works overnight before it dries, and something new appears on the wall by morning. Fresco the tool does the same thing. A graffiti writer working in secret at night, leaving something visual behind for people to discover. The wall is a fresco. The tag says "Fresco."

---

## The problem it solves

Static hero images get stale. Rotating them manually is tedious. AI image generation is powerful but requires infrastructure, scheduling, and storage to use continuously. Fresco handles all of that in a single CLI tool.

```bash
fresco generate                              # create an image
fresco upload $(fresco generate)             # create and publish
fresco remote copy dated.jpg latest.jpg      # update the stable URL
```

That's it. The image URL never changes. The image does.

## Phase 1 — CLI (current)

**What it is:** A Swift CLI tool that runs locally or via GitHub Actions.

**How it works:**
- `fresco generate` calls Gemini Imagen and writes the image locally
- `fresco upload` publishes a local image to Cloudflare R2
- `fresco remote copy` aliases a dated image to a stable filename (e.g. `latest.jpg`)
- GitHub Actions composes these commands on a daily cron schedule
- The project's README points to a stable R2 URL

**Who it's for:** Developers with GitHub repos who want daily images without running any server infrastructure.

**Stack:**
- Swift 6 + swift-argument-parser
- apple/swift-configuration for all config (environment variables)
- Google Gemini Imagen API
- Cloudflare R2 for image storage
- GitHub Actions for scheduling

**What it doesn't solve:** Projects without a GitHub repo — a Shopify store, a hosted website, anything that can't run GitHub Actions. That's Phase 2.

---

## Phase 2 — Self-hosted server

**What it is:** A Hummingbird server that runs on your own infrastructure and manages feeds with a persistent scheduler.

**How it works:**
- You run the Fresco server on a VPS (Hetzner, DigitalOcean, etc.)
- `fresco init --server https://your-fresco-instance.com` registers the project with your server
- The server handles scheduling — no GitHub Actions needed
- Any project, repo or not, can use a feed
- The CLI is identical — it just talks to the server instead of Gemini and R2 directly

**Who it's for:** Developers who want to run Fresco for multiple projects including non-GitHub ones, or who want a single place to manage all their feeds.

**Stack:**
- Everything from Phase 1
- Hummingbird 2.x HTTP server
- PostgreSQL for persistence
- Docker Compose on Hetzner or equivalent

**Open source:** The server is fully open source. Deploy it yourself, own your data, bring your own Gemini key and R2 bucket.

---

## Phase 3 — Hosted service

**What it is:** Phase 2 running at scale as a managed service. The Mastodon model — self-host if you want, or use ours if you don't.

**How it works:**
- Sign up at fresco.argylebits.com (or equivalent)
- Run `fresco init --server https://fresco.argylebits.com`
- Bring your own Gemini API key
- Everything else is managed

**Who it's for:** Anyone who wants daily AI images without touching a server. Designers, marketers, indie hackers, open source maintainers.

**What changes from Phase 2:**
- User accounts and authentication
- Billing and usage limits
- Onboarding flow
- Landing page and marketing

**What doesn't change:**
- The CLI — identical in all three phases
- The server codebase — Phase 3 IS Phase 2, just operated at scale
- The open source option — self-hosting is always available

---

## Design principles

**The CLI is the constant.** Commands and config never change between phases. A project initialized in Phase 1 works in Phase 3 with a single config line change.

**Credentials live in the environment.** API keys and secrets are environment variables — in `.env` locally (gitignored), in GitHub Actions secrets for CI. Never committed to the repo.

**Images live outside the repo.** Cloudflare R2 throughout all phases. No git history bloat, no LFS, no GitHub release hacks.

**Open source first.** The tool and the server are MIT licensed. The hosted service is a convenience layer on top of open infrastructure, not a lock-in play.

**Dogfooded from day one.** Fresco's own README banner and gallery are generated by Fresco itself. Every feature gets used by the project before it ships.
