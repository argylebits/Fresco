# Fresco

**AI-generated images for any project.**

Fresco is a composable CLI for generating images with the Google Gemini Imagen API and publishing them to Cloudflare R2. Drop the URL into any README, website, or app — it updates itself.

![Daily Fresco](https://pub-placeholder.r2.dev/fresco/latest.png)

---

## How it works

```bash
# Generate an image locally
fresco generate

# Generate and upload to R2
fresco upload $(fresco generate)

# Generate, upload, then alias to a stable URL (preserves extension)
IMAGE=$(fresco generate)
fresco upload "$IMAGE"
fresco remote copy "$(basename "$IMAGE")" "latest.${IMAGE##*.}"
```

Your project points at the stable URL:

```markdown
![Daily Image](https://pub-xxxx.r2.dev/your-project/latest.png)
```

A scheduled workflow generates, uploads, and copies to `latest.png` — the URL never changes.

## Install

```bash
brew install argylebits/tap/fresco
```

## Quick start

```bash
# Set up a project for daily images
cd your-project
cp fresco.template.env .env
# fill in your values

# Generate an image
fresco generate

# Generate and upload
fresco upload $(fresco generate)
```

## Phases

Fresco is being built in three phases:

**Phase 1 — CLI** ← you are here
Local tool. GitHub Actions for scheduling. Gemini + R2 direct.

**Phase 2 — Self-hosted server**
Persistent scheduler for non-GitHub projects. Run your own Fresco server.

**Phase 3 — Hosted service**
Sign up and use Fresco without running any infrastructure.

The CLI works the same way in all three phases. Phase 2 and 3 just change what's behind it.
